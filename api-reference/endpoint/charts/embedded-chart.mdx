---
title: "Get Embedded Chart"
api: "GET /chart/{share_string}/embedded"
description: "Retrieve chart data for embedding with support for public charts and SharePolicy tokens"
---

This endpoint returns chart data optimized for embedding. It supports both legacy public charts and modern SharePolicy-protected charts with signed tokens.

## Parameters

<ParamField path="share_string" type="string" required>
  The share string or chart ID for the embedded chart. For legacy charts, this can be the numeric chart ID. For modern charts, this is the unique share string.
</ParamField>

<ParamField query="token" type="string">
  Access token for SharePolicy-protected charts. Required if the chart has a SharePolicy with non-public visibility.
</ParamField>

<ParamField query="theme" type="string">
  Theme for the embedded chart display
  <ParamField body="theme" type="string">
    <ParamField body="light" type="string">Light theme</ParamField>
    <ParamField body="dark" type="string">Dark theme</ParamField>
    <ParamField body="os" type="string">System default theme</ParamField>
  </ParamField>
</ParamField>

## Response

Returns a Chart object with embedded-specific formatting:

<ResponseField name="id" type="string">
  The chart identifier
</ResponseField>

<ResponseField name="name" type="string">
  The chart title/name
</ResponseField>

<ResponseField name="type" type="string">
  The chart type (line, bar, pie, etc.)
</ResponseField>

<ResponseField name="subType" type="string">
  The chart subtype if applicable
</ResponseField>

<ResponseField name="chartData" type="object">
  The chart configuration and data optimized for rendering
</ResponseField>

<ResponseField name="chartDataUpdated" type="string">
  ISO 8601 timestamp of when the chart data was last updated
</ResponseField>

<ResponseField name="public" type="boolean">
  Whether the chart is publicly accessible
</ResponseField>

<ResponseField name="shareable" type="boolean">
  Whether the chart has sharing enabled
</ResponseField>

## Access Control

### Public Charts (Legacy)
- Charts with `public: true` can be accessed without tokens
- Only requires the chart ID or share string
- No additional authentication needed

### SharePolicy Charts
- Charts with SharePolicy require valid tokens unless visibility is "public"
- Token must be included in the `token` query parameter
- Token verification includes expiration and chart ID validation

### Error Responses

<ResponseField name="401" type="error">
  **Not authorized** - Chart is private or token is invalid/missing
</ResponseField>

<ResponseField name="400" type="error">
  **Bad request** - Invalid share string or malformed request
</ResponseField>

<ResponseField name="413" type="error">
  **Payload too large** - Chart data exceeds size limits
</ResponseField>

## Usage Notes

- **Rate Limiting**: This endpoint has rate limiting (50 requests per minute) to prevent abuse
- **Caching**: Chart data may be cached for performance - use cache-busting techniques if needed
- **Responsive**: The returned chart configuration is optimized for responsive embedding
- **CORS**: This endpoint supports CORS for cross-origin embedding
- **Legacy Support**: Numeric chart IDs are still supported for backwards compatibility
